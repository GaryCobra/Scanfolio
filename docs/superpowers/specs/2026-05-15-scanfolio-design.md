# Scanfolio — 同花顺股票截图扫描分析 App 设计文档

## 概述

Scanfolio 是一款纯离线 Android App，通过 OCR 识别同花顺股票列表截图，自动提取股票代码、名称及各数据列，并支持多张截图按股票代码自动拼接合并。用户可手动记录每笔交易的买入/卖出信息、标记战法成败，并基于成功/失败分组对各数据维度进行对比分析。

## 技术栈

- **语言/UI**：Kotlin + Jetpack Compose
- **本地存储**：Room（SQLite）
- **OCR 引擎**：Tesseract OCR（离线，中文语言包）
- **图表**：MPAndroidChart（或 Compose 原生图表库）
- **最低 SDK**：Android 8.0 (API 26)
- **网络权限**：不申请，完全离线

## 架构模块

App 分为 4 个核心模块，职责清晰：

```
截图识别模块  →  持仓管理模块  →  数据分析模块
      ↓              ↓
           设置管理模块
```

所有模块共享 Room 数据库，无网络依赖。

### 1. 截图识别模块

核心流程：用户导入/拍摄截图 → 图像预处理 → 表头识别 → 行分割 → OCR 识别 → 数据匹配与入库

**图像预处理**：灰度化 + 二值化，检测表格区域边界，按行切割。

**区域识别**：
- 第一行提取表头 → 识别当前截图包含哪些数据列（与 column_definitions 匹配）
- 每行第一列 = 代码+名称，其余列按表头顺序填入

**数据匹配**：
- 用 code+name 作为匹配键
- 已存在：只更新本次截图包含的列（按列级别合并，以最新截图时间为准）
- 不存在：新建 stock_records 记录

### 2. 持仓管理模块

底部导航 Tab 1。

- 股票列表：展示所有已识别股票，显示代码+名称+关键数据列（可配置默认展示列）
- 股票详情页：
  - 顶部：该股票所有数据列完整展示（动态排列）
  - 中部：关联的交易记录列表
  - 底部：添加交易 → 填写表单（买入时间/价格、卖出时间、获利比例/金额、战法、是否成功）
  - 持仓时间自动计算（sell_time - buy_time）
  - 支持编辑/删除交易记录
  - 支持手动修正 OCR 识别错误

### 3. 数据分析模块

底部导航 Tab 3，顶部三个子 Tab。

- **成功分析**：筛选标记为成功的交易，展示各数据列统计（均值、分布、区间筛选）
- **失败分析**：筛选标记为失败的交易，同上
- **对比分析**：成功 vs 失败两组并排对比，柱状图等图表展示差异

数值型列支持：均值对比、区间筛选（大于/小于/范围）
类别型列支持（所属概念、所属行业）：词频分布

### 4. 设置管理模块

侧边栏或悬浮按钮入口。

- **数据列管理**：增删列、排序、启用/禁用、指定类型（数值/百分比/文本）
- **战法管理**：增删战法名称，默认"爆量"
- **数据导出/导入**：全部数据（含配置）导出为 JSON 文件，可恢复

## 数据模型

### stock_records（股票基础信息）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, auto) | 自增主键 |
| code | String | 股票代码 |
| name | String | 股票名称 |
| data_columns | JSON String | 动态列键值对 |
| last_screenshot | Long | 最近截图时间戳 |
| created_at | Long | 创建时间 |

### trade_records（交易记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, auto) | 自增主键 |
| stock_record_id | Long (FK) | 关联 stock_records.id |
| buy_time | Long | 买入时间戳 |
| buy_price | Double | 买入价格 |
| sell_time | Long? | 卖出时间戳（可空） |
| holding_duration | Long? | 持仓时间（秒，自动计算） |
| profit_ratio | Double? | 获利比例（%） |
| profit_amount | Double? | 获利金额 |
| strategy_name | String | 战法名称 |
| is_success | Boolean | 是否成功 |
| created_at | Long | 创建时间 |

### strategy_types（战法类型）
| 字段 | 类型 | 说明 |
|------|------|------|
| name | String (PK) | 战法名称（如"爆量"） |
| created_at | Long | 创建时间 |

### column_definitions（数据列定义）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, auto) | 自增主键 |
| name | String | 列名（如"最新价"） |
| type | String | 数据类型：number / percentage / text |
| sort_order | Int | 排序序号 |
| enabled | Boolean | 是否启用 |
| created_at | Long | 创建时间 |

## 错误处理与边界情况

- **OCR 识别不准确**：预览页可逐行修正，置信度过低的行标红提示
- **新截图列名未定义**：弹窗提示用户添加到 column_definitions
- **截图无表头**：提示用户手动指定列顺序
- **数据冲突**：按列级别以最新截图时间为准覆盖
- **非表格截图**：结构异常时提示，不强行入库

## 离线/无网络约束
